### 디자인 패턴이란 무엇인가요?
디자인 패턴은 반복적으로 발생하는 설계 문제를 해결하기 위한 검증된 설계 방법입니다. <br>
즉, 객체지향 프로그래밍에서 자주 마주치는 구조적 문제를 효율적으로 해결하기 위한 공통적인 설계 템플릿이라고 할 수 있습니다.

심화 : <br>
디자인 패턴은 단순히 "형태"가 아닌 개발 의도와 구조적 원리의 집합입니다. <br>
잘 적용돤 디자인 패턴은 유지보수성을 높이고, 팀 간의 커뮤니케이션을 원활하게 하며, 코드의 재사용성을 극대화합니다. <br>

---
### 디자인 패턴은 어떤 종류로 나뉘나요?
디자인 패턴은 크게 세 가지 범주로 나뉩니다. <br>
객체를 생성하는 방법을 다루는 생성 패턴, 객체 간 관계를 구성하는 구조 패턴 그리고 객체 간의 협력을 다루는 행동 패턴입니다.

심화 : <br>
이 세 가지 분류 안에는 총 23개의 GoF 패턴이 존재하며, 대표적으로는 싱글톤, 팩토리 메서드, 어댑터, 옵저버 패턴 등이 있습니다. <br>
각 패턴은 특정 상황에서 발생하는 문제를 해결하기 위해 고안되었으며, 적절한 패턴을 선택하고 적용하는 것이 중요합니다. <br>

### 싱글톤 패턴이란 무엇인가요?
싱글톤 패턴은 하나의 클래스에 단 하나의 인스턴스만 존재하도록 보장하고, 어디서든 동일한 객체를 사용할 수 있게 하는 패턴입니다. <br>

심화 : <br>
싱글톤 패턴은 전역 상태를 관리하거나, 리소스 집약적인 객체를 효율적으로 재사용할 때 유용합니다. <br>
자바에서는 private 생성자와 static 메서드를 활용해 싱글톤을 구현합니다. <br>
```java
public class Singleton {
    private static Singleton instance; 
    private Singleton() {}
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

### 팩토리 메서드 패턴이란 무엇인가요?
팩토리 메서드 패턴은 객체 생성 코드를 별도의 클래스나 메서드로 분리해 캡슐화하는 방식입니다. <br>
즉, 객체 생성을 직접 하지 않고 팩토리 메서드를 통해 생성함으로써, 코드의 유연성과 확장성을 높입니다. <br>

심화 : <br>
클라이언트 코드가 구체적인 이름을 몰라도 객체를 만들 수 있어, 결합도를 낮출 수 있습니다. <br>
예를 들어, 다양한 제품을 생성하는 팩토리 클래스를 만들어, 새로운 제품이 추가될 때 기존 코드를 수정하지 않고도 확장할 수 있습니다. <br>


---
### 전략 패턴은 어떤 경우에 사용되나요?
존략 패턴은 행동을 인터페이스로 정의하고 런타임에 교체할 수 있게 구성하는 패턴입니다. <br>
즉 같은 기능을 수행하지만 다양한 방식이 존재할 때, 실행 중에 전략을 바꿔 사용할 수 있습니다. <br>

심화 : <br>
예를 들어 정렬 알고리즘을 전략 패턴으로 구현하면, 사용자가 원하는 정렬 방식을 선택해 적용할 수 있습니다. <br>
이로 인해 코드의 유연성이 증가하고, 새로운 전략을 추가할 때 기존 코드를 수정하지 않아도 됩니다. <br>
